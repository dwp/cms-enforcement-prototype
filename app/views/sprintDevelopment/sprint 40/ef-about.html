About

View parent finances (VPF) uses data from CMS2012.

Limited access to VPF

You can only access VPF from within CMS2012. You must have a paying parent selected. The button on the Contact Summary screen is available only for:

  • case workers with permission to use the service
  • paying parent cases
  • paying parents who pay one receiving parent

How VPF differs from CMS2012

We have used simple language and avoided jargon. Labels on the page are self-explanatory and use the language case workers would use to explain the details to parents.

Values shown on the Finances page

For cases that have previously been on Direct Pay for periods of time, CMS2012 includes collection charges that have beeen automatically satisfied in the total collection charges. VPF does not include these automatically satisfied collection charges. The values you see will be exactly what the parent has paid or was expected to pay.

Details about charges and debt on the Finances page

In CMS2012 you had to go to many screens to find information about charges and debt. In VPF we’ve provided a clear breakdown of charges on the Finances page so you can see exactly what charges or debt applied to the case. This charges and debt detail appears only if the parent is expected to pay these specific charges.

Cases page shows no qualifying children

If there are no children’s names visible this may be because there are no active children on the case.
